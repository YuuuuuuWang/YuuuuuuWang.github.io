LeetCode Rust-数组二分查找

寻找两个正序数组的中位数

给定两个大小为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的中位数。

进阶：你能设计一个时间复杂度为 O(log (m+n)) 的算法解决此问题吗？

示例 1：

输入：nums1 = [1,3], nums2 = [2]

输出：2.00000

解释：合并数组 = [1,2,3] ，中位数 2

示例 2：

输入：nums1 = [1,2], nums2 = [3,4]

输出：2.50000

解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5

示例 3：

输入：nums1 = [0,0], nums2 = [0,0]

输出：0.00000

示例 4：

输入：nums1 = [], nums2 = [1]

输出：1.00000

示例 5：

输入：nums1 = [2], nums2 = []

输出：2.00000

提示：

nums1.length == m
nums2.length == n
0 <= m <= 1000
0 <= n <= 1000
1 <= m + n <= 2000
-106 <= nums1[i], nums2[i] <= 106

思路

这道题最直接的思路是将两个数组合并为一个有序数组，然后返回其中位数。但是这样的时间复杂度会达到 O(m+n)，无法达到进阶要求的 O(log(m+n))。

因此，我们需要采用更为高效的算法。这里介绍一种经典的解法——二分查找（Binary Search）。

首先，我们将两个数组合并为一个虚拟的数组，这个虚拟数组中存放的是原始数组中所有数字按照从小到大排序后的结果。接着，我们需要找到这个虚拟数组中的中位数。

如果虚拟数组的长度是奇数，那么中位数就是这个数组中间的那个元素；如果虚拟数组的长度是偶数，那么中位数就是这个数组中间两个元素的平均值。

为了实现 O(log(m+n)) 的时间复杂度，我们可以使用二分查找的思想。具体来说，我们在虚拟数组中找到中间的位置，并比较该位置上的两个元素大小。如果该位置上的第一个数组的元素比第二个数组上的元素大，说明中位数位于第一个数组的左半部分或者第二个数组的右半部分。因此，我们可以抛弃第一个数组的右半部分或者第二个数组的左半部分，进一步缩小搜索区间。反之，如果该位置上的第一个数组的元素比第二个数组上的元素小，则说明中位数位于第一个数组的右半部分或者第二个数组的左半部分。同样可以抛弃对应的搜索区间。

根据这个思路，我们可以写出如下的 Rust 代码实现：

代码实现

struct Solution {}

impl Solution {
    pub fn find_median_sorted_arrays(nums1: Vec<i32>, nums2: Vec<i32>) -> f64 {
        let (m, n) = (nums1.len(), nums2.len());
        if m > n {
            return Solution::find_median_sorted_arrays(nums2, nums1);
        }
        let mut left = 0;
        let mut right = m;

        // 计算中位数在虚拟数组中的位置 k
        let k = (m + n + 1) / 2;

        while left < right {
            let mid1 = left + (right - left) / 2;
            let mid2 = k - mid1;

            // 检查是否越界
            let num1_left = if mid1 == 0 { i32::MIN } else { nums1[mid1 - 1] };
            let num1_right = if mid1 == m { i32::MAX } else { nums1[mid1] };
            let num2_left = if mid2 == 0 { i32::MIN } else { nums2[mid2 - 1] };
            let num2_right = if mid2 == n { i32::MAX } else { nums2[mid2] };

            // 比较大小，进一步缩小搜索区间
            if num1_left <= num2_right && num2_left <= num1_right {
                let max_left = num1_left.max(num2_left) as f64;
                let min_right = num1_right.min(num2_right) as f64;
                if (m + n) % 2 == 0 {
                    return (max_left + min_right) / 2.0;
                } else {
                    return max_left;
                }
            } else if num1_left > num2_right {
                right = mid1;
            } else {
                left = mid1 + 1;
            }
        }

        // 如果查找失败，说明输入的数组不是有序的
        panic!("Invalid input");
    }
}

测试用例

fn main() {
    // 测试用例
    let nums1 = vec![1, 3];
    let nums2 = vec![2];
    assert_eq!(Solution::find_median_sorted_arrays(nums1, nums2), 2.0);

    let nums3 = vec![1, 2];
    let nums4 = vec![3, 4];
    assert_eq!(Solution::find_median_sorted_arrays(nums3, nums4), 2.5);

    let nums5 = vec![0, 0];
    let nums6 = vec![0, 0];
    assert_eq!(Solution::find_median_sorted_arrays(nums5, nums6), 0.0);

    let nums7 = vec![];
    let nums8 = vec![1];
    assert_eq!(Solution::find_median_sorted_arrays(nums7, nums8), 1.0);

    let nums9 = vec![2];
    let nums10 = vec![];
    assert_eq!(Solution::find_median_sorted_arrays(nums9, nums10), 2.0);
}

注意：该算法并不能处理数组中存在重复元素的情况，如果存在重复元素，需要对算法进行适当的修改。
